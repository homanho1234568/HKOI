第六章｜資料結構與除錯技巧

熟悉堆疊、佇列、集合／字典等工具，加上系統化的除錯流程，能讓你在紙上與 IDE 之間迅速定位問題。

堆疊（Stack）與佇列（Queue）

堆疊

# 判斷括號是否匹配
stack = []
for ch in input().strip():
    if ch in \"([{\": stack.append(ch)
    else:
        if not stack: print(\"NO\"); break
        top = stack.pop()
        if top + ch not in (\"()\", \"[]\", \"{}\"):
            print(\"NO\")
            break
else:
    print(\"YES\" if not stack else \"NO\")

佇列

from collections import deque

q = deque()
for _ in range(int(input())):
    cmd = input().split()
    if cmd[0] == \"push\":
        q.append(cmd[1])
    else:
        print(q.popleft())

集合與字典

集合（set）用於快速去重、判斷是否出現過；字典（dict / map）則能儲存鍵值對。

# 統計單詞出現次數，並按照次數遞減輸出
from collections import Counter
words = input().split()
counter = Counter(words)

for word, cnt in counter.most_common():
    print(word, cnt)
考試常見：統計頻率、判斷是否出現過、建立對照表（例如 char → index）都離不開 set 與 dict。

快速定位 bug

  • 列印關鍵狀態：在迴圈內輸出索引、當前值或暫存變數，確認是否符合預期。
  • 切分問題：把函式拆開測試，先確認輸入處理正確，再檢查邏輯。
  • 極端測資：輸入最小值（例如 0 或 1）、最大值、全部相同或完全遞減的資料。

當遇到 Wrong Answer 時，先復刻評測平台的 Sample Input，若 Sample 都過不了，就代表格式或流程仍有漏洞。

練習建議

  1. 使用堆疊判斷多種括號是否匹配，並輸出第一次出錯的位置。
  2. 佇列模擬排隊系統：支援 push、pop、front 查詢與大小查詢。
  3. 讀入一段文字，輸出前 K 個最常出現的單字（按照頻率與字典序排序）。

錯題完成後可再次回到題庫中心，或挑戰歷屆卷一乙部涉及資料結構的題目。
