第五章｜排序與複雜度

懂得選擇適合的排序方法與估算複雜度，能大幅提升題目的效率與可讀性。

內建排序與 key

Python 與 C++ 都有高效的排序工具，善用 key / comparator 可以少寫很多程式碼。

# 排序學生 (分數遞減，名字遞增)
students = [
    ("Iris", 92),
    ("Ben", 95),
    ("Calvin", 95)
]
students.sort(key=lambda x: (-x[1], x[0]))
print(students)
注意：多重排序時可以用「先排序次要條件，再排序主要條件」，或像上例一樣利用 key。

手寫排序與應用

有時題目會要求你實作排序（例如只能用交換或只能比對部分元素），需要了解基本算法。

# 選擇排序
arr = list(map(int, input().split()))
n = len(arr)
for i in range(n):
    min_idx = i
    for j in range(i + 1, n):
        if arr[j] < arr[min_idx]:
            min_idx = j
    arr[i], arr[min_idx] = arr[min_idx], arr[i]
print(arr)

雖然效能不如內建 sort，但有助於理解比較與交換的過程，必要時也能修正為部分排序。

複雜度概念

算法 平均時間複雜度 備註
Bubble / Selection / Insertion O(n2) 資料量小或幾乎已排序時尚可
Merge / Quick / Timsort O(n log n) 一般建議直接使用內建排序
Counting / Bucket / Radix O(n + k) 適用於資料範圍固定且不大時
估算技巧：當題目資料上限約 105 時，O(n log n) 通常仍可接受；若達到 106 以上，需搭配更快的演算法或 I/O 優化。

練習建議

  2. 給定 N 個座標，按照距離原點的距離排序，距離相同時再依角度排序。
  3. 實作 counting sort，處理分數介於 0～100 的資料，並輸出每個分數出現次數。
